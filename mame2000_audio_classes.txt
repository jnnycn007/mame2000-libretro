mame2000 per-game audio class list
======================================================================

WHAT THE CLASSES MEAN
----------------------------------------------------------------------
Class 1 (no resampling): every sound source in the game synthesizes audio
  directly at the core's output rate, so the mixer is a 1:1 copy and there
  is no rate conversion inside the core. (AY8910, SN76496, YM2203, YM2612,
  YM2608, YM2610/B, OKIM6295, DAC, MSM5205/ADPCM, QSOUND, SEGAPCM,
  K-series PCM, POKEY, NES, TIA, ASTROCADE, UPD7759, HC55516, etc.)

Class 2 (fixed native rate): the source runs at a fixed rate derived from
  its own clock, not the output rate. (YM2151, YM3812/YM3526/Y8950/YM2413,
  C140, VLM5030, NAMCO.)

Per game we bucket as:
  NONE     - 0 class-2 sources             -> already no core resampling
  SINGLE   - exactly 1 class-2 native rate -> output AT that rate, resample-free
  MULTI    - >=2 distinct class-2 rates    -> some resampling unavoidable
  SPECIAL  - SAMPLES/CUSTOM only           -> rate is runtime/driver dependent
  SINGLE + SAMPLES/CUSTOM
           - 1 class-2 chip but SAMPLES/CUSTOM is also present so
             game_fixed_output_rate() short-circuits to 0 and the
             configured rate is used.

RATE POLICY
----------------------------------------------------------------------
SINGLE games output at the chip's own fixed rate so the core never resamples;
the frontend performs the single conversion to the host rate. The rate is
derived from the chip clock:
  YM2151                           = clk / 64
  YM3812 / YM3526 / Y8950 / YM2413 = clk / 72
  C140                             = interface frequency
  VLM5030                          = clk / 440
  NAMCO                            = base rate doubled to 192000 then /4

NONE games are already resample-free at any rate. MULTI, SPECIAL and
SINGLE+SAMPLES/CUSTOM games follow the 'Sample Rate (Hint)' core option.

Chips that were class-2 in mame2003 but do NOT appear in mame2000 (the
codebase predates them): YMF262, NAMCONA, SP0250, SCSP, GAELCO_GAE1,
GAELCO_CG1V.

DIFFERENCES FROM MAME2003
----------------------------------------------------------------------
The set of class-2 chips this core considers, the per-chip rate formula
and the bucketing rules above mirror mame2003 exactly. Per-game
buckets sometimes differ because the underlying drivers do: an
mame2000 driver may declare a different chip set than its mame2003
descendant (newer revisions add chips or replace customs with the
class-2 equivalent), and a handful of newer games are absent
altogether.
